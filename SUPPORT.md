# Mediabot v3 support

Thank you for using Mediabot v3.

Please choose the appropriate support channel so that questions, bugs and suggestions can be handled efficiently.

## Report a reproducible bug

Open a [GitHub Issue](https://github.com/teuk/mediabot_v3/issues) and select the **Bug report** template.

Please include:

* your Mediabot version;
* your operating system and Perl version;
* your MariaDB or MySQL version;
* the IRC network concerned;
* clear steps to reproduce the problem;
* the expected and actual behavior;
* relevant sanitized logs.

Never publish passwords, API keys, IRC credentials, database credentials, cookies, tokens or other private information.

## Suggest an improvement

Open a [GitHub Issue](https://github.com/teuk/mediabot_v3/issues) and select the **Feature request** template for a concrete and clearly described improvement.

Feature requests may include:

* new IRC commands;
* improvements to existing commands;
* plugins or scripting features;
* Partyline and administration features;
* URL or media handling;
* radio features;
* metrics, monitoring or logging;
* installation and configuration improvements;
* documentation and testing.

For early ideas or proposals that still need discussion, use [GitHub Discussions](https://github.com/teuk/mediabot_v3/discussions) instead.

## Installation and usage questions

Use [GitHub Discussions](https://github.com/teuk/mediabot_v3/discussions) for:

* installation help;
* configuration questions;
* usage questions;
* general troubleshooting;
* ideas that are not yet ready to become feature requests.

Please search existing Issues and Discussions before creating a new request.

## Live IRC community chat

Connect to the **EpiKnet** IRC network using:

* **Server:** `irc.epiknet.org`
* **Port:** `6697`
* **Encryption:** SSL/TLS
* **Channel:** `#i/o`

Join `#i/o` for live discussion, community help and quick troubleshooting.

IRC conversations are useful for diagnosis and discussion, but confirmed bugs should also be recorded in GitHub Issues so they can be tracked and fixed.

## Security issues

Do not report security vulnerabilities or exposed credentials in a public GitHub Issue or Discussion.

Use GitHub's private vulnerability reporting feature and see `.github/SECURITY.md` for the complete security reporting procedure.

## Contributions

Contributions are welcome, including:

* bug fixes;
* documentation corrections;
* tests;
* plugins;
* command improvements;
* installation improvements;
* translations;
* code review and testing on additional IRC networks.

See `CONTRIBUTING.md` for the complete contribution guidelines.
